# clean-room-sim

Companion repository for the manuscript

> **cleanTMLE: An R Package for Staged, Outcome-Blind Targeted Learning
> with Plasmode Data-Quality Stress Testing**

The repository contains the manuscript, the simulation study that
produces its results, and a git submodule with the
[`cleanTMLE`](cleanTMLE/) R package itself.

## What the project does

The package and the simulation together demonstrate three contributions:

1. **A staged, outcome-blind TMLE workflow in R.** `cleanTMLE`
   implements the staging-and-clean-room construct of Muntner et al.
   (2024, *Pharmacoepidemiology and Drug Safety*), with hashed
   analysis locks, an audit trail, a decision log, and Stage-4
   estimators that physically refuse to run on a lock whose outcome
   column has not been authorised by the gate.
2. **A plasmode data-quality stress test.** `run_plasmode_dq_stress()`
   extends the standard outcome-blind plasmode loop with four
   degradation mechanisms (covariate missingness, treatment
   misclassification, outcome misclassification, unmeasured
   confounding) and produces a degradation gradient — bias, RMSE,
   and CI coverage at increasing severities — for each prespecified
   TMLE candidate. The gradient lets the GO/FLAG/STOP gate score the
   candidate against the plausible range of every threat documented
   in SPIFD2 (Gatto et al., 2023).
3. **A Monte Carlo evaluation.** [`run_simulation.R`](run_simulation.R)
   compares six estimators (crude, IPTW, PS matching, TMLE,
   cross-validated TMLE, matched-cohort TMLE) under three scenarios
   (good overlap, marginal overlap, unmeasured confounding) and
   exercises the full Stage 1a → Stage 4 pipeline including
   checkpoints, audit log, decision log, and the DQ stress test.

The `cleanTMLE` package reuses the cohort-specification grammar and
several reporting helpers from the
[`causalRisk`](https://github.com/CausalInference/causalRisk)
package (Brookhart, Breskin, Eledath; MIT-licensed) under
attribution. `cleanTMLE` adds the lock, the gate, the audit trail,
the modular TMLE primitives, and the plasmode DQ stress test that
`causalRisk` does not provide.

## Reproducing the results

### 1. Install dependencies

```r
install.packages(c(
  "devtools", "ggplot2", "rlang", "sandwich", "survival",
  "SuperLearner", "glmnet", "tmle", "MatchIt", "yaml",
  "knitr", "rmarkdown", "quarto"
))
```

`cleanTMLE` itself is installed from the submodule:

```r
devtools::install("cleanTMLE")
```

### 2. Run the simulation

```bash
Rscript run_simulation.R
```

Default settings: `n = 2000` per replicate, 200 Monte Carlo
replicates per scenario, 30 plasmode replicates for candidate
selection, 30 plasmode replicates per DQ scenario level. Runtime
is roughly six to eight hours on a modern laptop. Progress is
saved every 10 replicates so the run can be monitored or resumed.

To monitor a running simulation in another terminal:

```r
source("check_progress.R")
```

### 3. Render the manuscript

```bash
quarto render reports/manuscript_outcome_blind_dq.qmd
```

This produces `reports/manuscript_outcome_blind_dq.html` from the
Monte Carlo and DQ-stress results in `results/`.

## Outputs the simulation produces

For each of three scenarios (`good_overlap`, `marginal_overlap`,
`unmeasured_conf`):

- `results/simulation_results.rds` — combined object with config,
  truths, scenarios, results, summaries, audit logs, and runtime.
- `results/summary_<scenario>.csv` — per-scenario summary table
  (estimator, bias, RMSE, coverage, SE/SD ratio, CI width).
- `results/plasmode_<scenario>.csv` — plasmode candidate-selection
  metrics.
- `results/dq_stress_<scenario>.rds` — DQ stress-test metrics.
- `results/audit_<scenario>.csv` and `decision_log_<scenario>.csv` —
  audit trail and decision log exports.
- `results/lock_<scenario>.rds` — final analysis lock.
- `results/interim_<scenario>.rds` — incremental Monte Carlo
  checkpoint, refreshed every 10 replicates.
